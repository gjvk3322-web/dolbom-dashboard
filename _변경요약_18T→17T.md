## ⚠️ 추가로 작업 필요한 시스템 (HTML 외)

이번 zip의 HTML만 배포하면 일부 기능에서 시트와 식별자가 안 맞아 오류 가능.
아래 작업도 함께 진행해야 일관성 유지됨.

### 1. Apps Script `Code.gs` (확장 클로드에 부탁)
변경할 위치 (메모리상 25곳):
- `'모던1m18'` → `'모던1m17'`
- `'베이지1m18'` → `'베이지1m17'`
- `'1000_18'` → `'1000_17'`
- `'1000매트 18T'` → `'1000매트 17T'`
- `'1000매트18T'` → `'1000매트17T'`
- `'18T'` 표시 텍스트 모두

특히 주의:
- `mapColorForVideo()` 함수 안 매핑 테이블
- `setupScheduleSheets()` 안 데이터 유효성 검사 리스트
- `createMarginSheet()`, `createSimulationSheet()`, `createScenarioSheet()` 안 라벨

### 2. AWS 서버 `inventory_check.py`
```bash
# 맥북 터미널
ssh -i dolbom-key.pem ubuntu@3.35.253.214

# AWS 서버 안에서
cd ~/돌봄매트/수집기/
cp inventory_check.py inventory_check.py.bak
sed -i 's/18T/17T/g; s/1m18/1m17/g; s/1000_18/1000_17/g' inventory_check.py
diff inventory_check.py.bak inventory_check.py | head -20
```

### 3. Google Sheets 기존 데이터
시트 메뉴 → 수정 → 찾기 및 바꾸기 (Cmd+Shift+H):
- 모든 시트 일괄 적용
- `1m18` → `1m17`
- `1000_18` → `1000_17`
- `18T` → `17T`

특히 시공스케줄(부산), 시공스케줄(경기) 시트의 `U열` (색상)
- 기존 데이터의 `'모던1m18'`, `'베이지1m18'`이 새 식별자와 안 맞으면 통계 깨짐

- 18T 표기된 템플릿 모두 17T로 수정
- 시공안내, 결제안내, 시공완료 등
